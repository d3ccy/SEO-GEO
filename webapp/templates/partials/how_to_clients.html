How to use this tool ▼

How to use

Click + New Client to create a profile. Fill in the Name (required), domain (without https://), project name, CMS, location, and any internal notes.

Once saved, the client appears in a dropdown on every tool page. Selecting a client auto-fills the form fields — saving time across all five tools. The more complete the profile, the more fields are auto-filled.

Use the Edit and Delete actions in the table to manage existing profiles.

What auto-fills where

GEO Audit
URL field → client domain (prefixed with https://)
Keyword Research
Seed keyword → domain, Location → client location
AI Visibility
Domain field → domain, Brand query → client name
Domain Overview
Domain field → domain, Location → client location
Content Guide
All fields: name, domain, project name, CMS

Quick tips

  • Always save a client profile before starting a project — it saves time across all five tools
  • The Notes field is for internal context (e.g. "Client uses Acquia hosting, check with Jon before Cloudflare changes")
  • Location defaults to United Kingdom (2826). Change this for clients with non-UK target audiences
  • All team members share the same client list — this is intentional for a shared team tool
  • Delete old profiles after projects conclude to keep the list manageable

Common questions

Will my clients be visible to other users of the app?
Yes. All users of the same app instance share one client list — the same way a shared spreadsheet would work.
I lost my client profiles after a redeploy — why?
Client profiles are stored in webapp/data/clients.json, which is not persisted between Railway free-tier redeploys. Ask the developer about adding a Railway Volume for persistent storage.
Can I export the client list?
Not from the UI currently. Ask the developer for a copy of clients.json — it's plain JSON readable in any text editor or importable into a spreadsheet.
